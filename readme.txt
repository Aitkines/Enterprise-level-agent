【文件夹名称】
素材与源码

【文件夹作用】
本文件夹用于存放作品源代码压缩包及智能体交互记录文件，供评审专家查看项目实现内容以及智能体在作品创作过程中的参与情况。

【文件内容说明】
1. 光之耀面项目源码.zip
该文件为作品源代码压缩包，包含“光之耀面”项目的核心源码及运行相关文件，用于展示作品的实际开发内容、系统结构和主要实现方式。

压缩包内主要内容说明如下：

（1）frontend
前端源码目录，包含 React 页面、组件、样式及前端构建配置，用于实现系统界面展示与交互。

（2）backend
后端接口目录，包含 FastAPI 接口层代码，用于提供聊天、会话管理、财务分析、公司对比、报告生成等服务接口。

（3）src
核心业务源码目录，包含业务服务层、基础设施层、共享模块及领域逻辑，是项目主要功能实现所在位置。

（4）static
静态资源目录，用于存放系统运行所需的静态文件和样式资源。

（5）utils
工具目录，用于存放项目运行或开发过程中使用的辅助脚本和通用处理函数。

（6）.streamlit
Streamlit 相关配置目录，用于兼容项目早期或备用界面运行方式。

（7）agent_engine.py
项目核心智能体引擎文件，用于组织研究助手提示词、模型调用及部分分析处理逻辑。

（8）api_server.py
后端服务启动入口文件，用于启动 FastAPI 服务。

（9）app.py
Streamlit 兼容启动入口文件，用于启动旧版或兼容界面。

（10）start_dev.ps1
Windows PowerShell 启动脚本，用于一键启动前后端开发环境，并可在首次使用时安装依赖。

（11）start_dev.bat
Windows 批处理启动脚本，用于封装 PowerShell 启动流程，便于双击运行。

（12）requirements.txt
Python 依赖清单文件，用于安装项目后端及数据分析相关依赖。

（13）docker-compose.yml
Docker 编排文件，用于通过容器方式统一启动前端和后端服务。

（14）Dockerfile.backend
后端 Docker 镜像构建文件，用于构建项目后端运行镜像。

（15）README.md
项目说明文档，包含项目简介、运行方式和结构说明等内容。

（16）.env.example
环境变量示例文件，用于说明项目运行所需的接口密钥及配置项。

（17）.gitignore 与 .dockerignore
分别用于控制版本管理忽略规则和 Docker 构建忽略规则。

2. 智能体交互记录.json
该文件为智能体交互记录文件，用于记录项目开发过程中与智能体的部分交互内容，体现智能体在需求理解、方案建议、代码修改、说明文档整理等环节中的协作过程。

【补充说明】
1. 源代码压缩包用于展示作品完整实现内容和系统主要结构。
2. 智能体交互记录文件用于说明智能体在作品创作过程中的辅助作用。
