# 系统部署说明（最新版）

## 1. 部署目标
本项目支持两种运行方式：

1. 本地开发运行（前后端分开）。
2. 固定域名公网演示运行（后端统一入口 + ngrok）。

当前答辩推荐使用第 2 种方式。

## 2. 运行环境要求

- Windows 10/11
- Python 3.10+
- Node.js 18+
- ngrok（已配置账号与 authtoken）

## 3. 本地开发运行

在项目根目录执行：

```powershell
.\start_dev.ps1
```

默认地址：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:8000`

## 4. 固定域名公网运行（推荐）

在项目根目录执行：

```powershell
.\start_public.ps1 -NgrokDomain aroma-espresso-patience.ngrok-free.dev
```

执行完成后，脚本会自动：

1. 启动后端服务。
2. 启动 ngrok 固定域名隧道。
3. 生成并更新运行网址与二维码文件。

## 5. 自动生成文件位置

运行后会自动更新以下文件：

1. `1_作品与答辩材料/运行网址.txt`
2. `1_作品与答辩材料/系统访问二维码.png`
3. `public_runtime/runtime_url.txt`
4. `public_runtime/runtime_qr.png`
5. `public_runtime/runtime_qr_note.txt`
6. `public_runtime/start_public.trace.log`

## 6. 访问说明

对外访问地址（固定域名）：

`https://aroma-espresso-patience.ngrok-free.dev`

若首次访问看到 ngrok 提示页，请点击 `Visit Site` 后继续访问。

## 7. 停止服务

可在 PowerShell 执行：

```powershell
Get-Process ngrok -ErrorAction SilentlyContinue | Stop-Process -Force
Get-Process python -ErrorAction SilentlyContinue | Stop-Process -Force
```

## 8. 重要说明

固定域名模式下，运行网址与二维码可保持一致。  
若不使用 `-NgrokDomain`，ngrok 可能分配新域名，网址与二维码需要重新生成。
