# 光之耀面系统部署说明

## 1. 系统简介

光之耀面（Radiant Surface）是一个面向企业研究、投研分析与智能问答场景的系统。项目采用前后端分离架构：

- 前端：React + Vite
- 后端：FastAPI
- 数据与分析能力：pandas、akshare、pdfplumber、langchain 等
- 模型接口：Volcengine Ark、Doubao Seed Lite

系统支持多轮对话、会话管理、公司研究、财务分析、对比分析、研究报告生成和 PDF 导出等功能。

## 说明

当前作品提交材料以“本地部署运行”作为主要演示方式。评审专家可按照本说明文档在本地环境中完成系统部署、启动与访问。

## 2. 部署环境要求

### 2.1 基础环境

- 操作系统：Windows、Linux 或 macOS
- Python：3.10 及以上
- Node.js：18 及以上
- npm：9 及以上

### 2.2 Python 依赖

项目后端依赖由根目录下的 `requirements.txt` 统一管理。

安装命令：

```powershell
pip install -r requirements.txt
```

### 2.3 前端依赖

前端依赖由 `frontend/package.json` 管理。

安装命令：

```powershell
Set-Location frontend
npm install
```

## 3. 项目目录说明

当前项目运行核心主要包括以下目录和文件：

- `frontend/`
  前端源码目录。

- `backend/`
  后端接口目录。

- `src/`
  核心业务逻辑目录。

- `agent_engine.py`
  智能体核心逻辑文件。

- `api_server.py`
  后端启动入口。

- `start_dev.ps1`
  Windows 下的一键启动脚本。

- `docker-compose.yml`
  Docker 编排文件。

## 4. 环境变量配置

项目运行前需在根目录准备 `.env` 文件，可由 `.env.example` 复制得到：

```powershell
Copy-Item .env.example .env
```

当前主要环境变量如下：

```env
ARK_API_KEY=your_ark_api_key_here
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
DOUBAO_SEED_LITE_KEY=your_doubao_seed_lite_key_here
DOUBAO_SEED_LITE_ENDPOINT=your_endpoint_id_here
```

变量说明：

- `ARK_API_KEY`：Volcengine Ark 平台访问密钥
- `ARK_BASE_URL`：Ark 平台接口地址
- `DOUBAO_SEED_LITE_KEY`：Doubao Seed Lite 访问密钥
- `DOUBAO_SEED_LITE_ENDPOINT`：Doubao Seed Lite 对应 endpoint

## 5. 本地部署与启动方式

### 5.1 推荐方式

Windows 下推荐使用项目自带脚本进行一键启动：

```powershell
.\start_dev.ps1
```

如首次运行，需要先安装依赖，可使用：

```powershell
.\start_dev.ps1 -InstallDeps
```

执行后将分别启动：

- 后端服务：`http://127.0.0.1:8000`
- 前端服务：`http://127.0.0.1:5173`

以上地址为当前作品本地部署演示时的默认访问地址。

### 5.2 手动启动方式

#### 1. 安装后端依赖

```powershell
pip install -r requirements.txt
```

#### 2. 启动后端服务

```powershell
python api_server.py
```

或使用：

```powershell
uvicorn backend.api.main:app --host 127.0.0.1 --port 8000
```

#### 3. 启动前端服务

```powershell
Set-Location frontend
npm install
npm run dev
```

## 6. Docker 部署方式

项目已提供 Docker 部署配置，可使用以下命令启动：

```powershell
docker compose up --build
```

启动后默认服务地址：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:8000`

Docker 部署相关文件：

- `docker-compose.yml`

## 7. 正式在线部署方式

当前项目可采用“前端部署到 Vercel、后端部署到 Render”的方式完成正式在线部署。

### 7.1 后端部署到 Render

项目根目录已提供 `render.yaml`，可用于创建 Render Web Service。

后端部署要点如下：

- 运行时：Python
- 构建命令：`pip install -r requirements.txt`
- 启动命令：`uvicorn backend.api.main:app --host 0.0.0.0 --port $PORT`
- 健康检查地址：`/api/health`

Render 侧需要配置的环境变量如下：

- `ARK_API_KEY`
- `ARK_BASE_URL`
- `DOUBAO_SEED_LITE_KEY`
- `DOUBAO_SEED_LITE_ENDPOINT`

部署完成后，Render 会生成一个后端公网地址，例如：

`https://your-backend-name.onrender.com`

### 7.2 前端部署到 Vercel

前端目录为 `frontend/`，部署时建议在 Vercel 项目设置中将 Root Directory 指向 `frontend`。

前端部署要点如下：

- Framework Preset：Vite
- Root Directory：`frontend`
- Build Command：`npm run build`
- Output Directory：`dist`

前端生产环境变量需要配置：

- `VITE_API_BASE`

其值应填写为 Render 后端公网地址，例如：

`https://your-backend-name.onrender.com`

项目中已提供前端生产环境变量示例文件：

- `frontend/.env.production.example`

部署完成后，Vercel 会生成一个前端公网地址，例如：

`https://your-frontend-name.vercel.app`

### 7.3 在线部署后的访问关系

在线部署完成后，访问方式如下：

- 用户访问 Vercel 提供的前端网址
- 前端通过 `VITE_API_BASE` 请求 Render 提供的后端接口
- 后端完成对话、财务分析、报告生成等处理后返回结果

### 7.4 说明

当前项目已具备正式在线部署所需的基本结构、依赖清单和部署配置文件。若需实际完成上线，还需在 Vercel 和 Render 平台分别创建项目并填写相应环境变量。

## 8. 主要功能入口说明

系统启动后，用户可通过前端页面访问以下主要功能：

- 对话与会话管理
- 公司研究问答
- 财务分析
- 公司对比分析
- 报告生成
- PDF 导出

后端主要接口包括：

- `GET /api/health`
- `GET /api/dashboard/overview`
- `GET /api/dashboard/system-status`
- `GET /api/sessions`
- `POST /api/chat`
- `POST /api/chat/stream`
- `GET /api/financial/{target}`
- `GET /api/comparison/{symbol}`
- `POST /api/report`
- `POST /api/report/pdf`

## 9. 注意事项

1. 运行前必须正确配置 `.env` 文件中的接口密钥。
2. 若前端无法连接后端，请检查后端服务是否已正常启动，并确认默认地址是否为 `http://127.0.0.1:8000`。
3. 若需部署为线上版本，应将前端环境变量 `VITE_API_BASE` 配置为实际的线上后端地址。
4. 当前比赛材料可采用本地部署方式进行展示与评审。
5. Render 免费实例在长时间无请求后可能出现冷启动，首次访问响应可能较慢。
6. 若线上服务需要长期保留本地会话数据，需进一步配置持久化存储方案。
7. 提交比赛材料时，建议将本说明文档导出为 PDF，并与部署包、运行地址说明和二维码说明文件一并提交。
