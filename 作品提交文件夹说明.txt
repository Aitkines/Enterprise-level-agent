【作品提交文件夹设计建议】

建议将作品提交文件夹设计为以下结构：

作品提交文件
├─ 1_作品与答辩材料
│  ├─ 可执行文档
│  │  ├─ 系统部署说明.pdf
│  │  ├─ 系统使用说明.pdf
│  │  ├─ 运行网址.txt
│  │  └─ 系统访问二维码.png
│  └─ 部署包
│     └─ 光之耀面部署包.zip
└─ 2_素材与源码
   └─ 光之耀面项目源码


【一、可执行文档部分写法】

文件夹名称：
可执行文档

文件夹作用：
本文件夹用于存放作品系统运行和使用所需的说明性材料，便于评审专家了解系统的部署方式、访问方式和操作方法。

文件内容说明：
1. 系统部署说明.pdf
用于说明系统所需运行环境、依赖配置、部署流程、启动步骤和注意事项，便于作品在其他设备或服务器环境中完成部署。

2. 系统使用说明.pdf
用于说明系统主要功能、页面结构、操作流程和使用方法，便于评审专家快速了解和体验作品功能。

3. 运行网址.txt
用于提供作品系统的在线访问地址。文件中建议写明系统名称、前端访问地址、后端接口地址（如需）、以及访问说明。

4. 系统访问二维码.png
用于通过扫码方式快速访问作品系统在线地址，便于展示、评审和现场答辩使用。


【二、部署包部分写法】

文件夹名称：
部署包

文件夹作用：
本文件夹用于存放作品系统的可部署运行文件，便于作品在目标环境中快速完成安装和运行。

文件内容说明：
1. 光之耀面部署包.zip
该压缩包用于提供作品系统运行所需的部署文件，建议包含后端运行代码、前端构建产物或前端工程文件、依赖清单、环境变量示例文件、部署配置文件及启动说明文档等内容。

部署包建议包含内容：
1. 后端运行代码
2. 前端工程文件或构建产物
3. requirements.txt
4. docker-compose.yml
5. Dockerfile.backend
6. .env.example
7. 启动说明文档


【三、运行网址文件建议写法】

文件名：
运行网址.txt

建议正文内容如下：

系统名称：光之耀面
系统访问地址：https://请填写你的线上访问地址
后端接口地址：https://请填写你的后端接口地址
访问说明：请使用现代浏览器访问系统前端地址，进入系统后可体验多轮对话、财务分析、公司对比、报告生成等功能。


【四、二维码文件说明写法】

文件名：
系统访问二维码.png

说明文字可写为：
本二维码对应作品系统线上运行网址，扫码后可直接访问系统首页，用于作品展示、专家评审和答辩演示。


【五、源码部分写法】

文件夹名称：
光之耀面项目源码

文件夹作用：
本文件夹用于存放作品源代码及运行配置文件，包含前端、后端、核心业务逻辑、依赖清单、启动脚本和说明文档，用于展示作品的完整实现内容。

源码部分说明可写为：
本文件夹包含“光之耀面”项目的完整源码及相关配置文件。其中包括前端界面代码、后端接口代码、业务逻辑代码、项目依赖文件、启动脚本和说明文档，可用于系统开发说明、功能实现展示和项目复现。


【六、可直接使用的简化版总说明】

1. 可执行文档
包含作品系统的部署说明、使用说明、运行网址及访问二维码等材料，用于说明系统如何部署、如何使用以及如何在线访问。

2. 部署包
包含作品系统运行所需的可部署文件，用于在目标环境中完成安装、部署和启动。

3. 运行网址
用于提供作品系统的线上访问地址，便于评审专家直接在线体验作品功能。

4. 二维码
用于通过扫码方式快速访问作品系统运行网址，便于展示和答辩使用。

5. 源码
包含作品完整源代码及相关配置文件，用于展示作品的实际开发内容和系统实现过程。
